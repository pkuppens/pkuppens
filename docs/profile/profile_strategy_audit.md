## 2. GitHub Profile Best Practices (Summary)

Sources: [GitHub Docs](https://docs.github.com/en/account-and-profile/how-tos/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme), [Dev Genius Guide](https://blog.devgenius.io/the-developers-guide-to-github-profile-readmes-that-stand-out-d45a9148bdee).

### Structure & Readability

- Clear headings and bullet points; short paragraphs
- Scannable layout: white space, dividers, strategic emojis
- Professional intro: who you are, what you do, current focus
- Skills/technologies: badges improve visual scan
- Call to action: invite visitors to connect or explore

### Repository Organisation

- Pin 4–6 strongest repos (on_prem_rag, babblr, healthcare-aigent, etc.)
- Ensure pinned repos have clear READMEs
- Profile README appears above pinned repos—first impression

### Profile Completeness

- **Bio**: Short tagline (GitHub About section)
- **Location**: Fills geographic filter for recruiters
- **Website**: Link to pieterkuppens.net
- **Social links**: LinkedIn, X/Twitter if applicable
- Empty About fields hurt search visibility

### Discoverability

- Recruiters filter by location, language, keywords
- Include "freelance", "Python", "AI", "healthcare", "remote" if accurate
- Bidirectional links (profile ↔ LinkedIn ↔ website) improve cross-platform discovery

## 3. SEO & Discoverability

- GitHub profiles are indexed by search engines; clear headings and keywords help
- Profile README content (intro, skills, projects) is crawlable
- Consistent messaging across GitHub, LinkedIn, and personal site reinforces credibility

## 4. Exemplar Comparison

Senior developers with strong profiles typically include:

- Value proposition or headline (what you offer)
- 2–4 current/highlight projects with tech badges
- Education and certifications (credibility)
- Clear availability/CTA
- Links to LinkedIn and personal site

## 5. Strengths and Gaps

### Strengths

- Technical tone appropriate for GitHub
- Current projects (RAG, Babblr) with badges and screenshots
- Skills library differentiates (AI-assisted workflow expertise)
- Certifications and career narrative present
- Concise; no bloat

### Gaps

| Gap | Impact |
|-----|--------|
| Education not visible | Credibility signal missing (TU/e, MSc, PDEng) |
| Availability buried in Interests | Prospects may miss freelance CTA |
| One-way links only | LinkedIn/website don’t yet link back to GitHub |
| No value proposition headline | "Complete department" positioning absent |
| Pinned repos may not be optimised | Check repo READMEs and order |
